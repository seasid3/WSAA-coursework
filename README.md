## Assignment 2 - Card Draw
**File:** `deckofcards.py`

### Description
A program that simulates dealing 5 cards from a shuffled deck using the [Deck of Cards API](https://deckofcardsapi.com/). It checks the hand for special combinations and congratulates the user.

### How it works
1. Calls the Deck of Cards API to shuffle a new deck and retrieve a deck ID
2. Draws 5 cards using the deck ID
3. Prints the value and suit of each card
4. Checks the hand for:
   - Pair
   - Three of a kind
   - Straight (5 consecutive values)
   - Flush (all same suit)

### How to run
```bash
python deckofcards.py
```

## Assignment 3 - CSO Data Retrieval
**File:** `assignment03-cso.ipynb`

### Description
A Jupyter notebook that retrieves the "Exchequer Account (Historical Series)" dataset from the Central Statistics Office (CSO) API and saves it to a local JSON file.

### How it works
1. Calls the CSO PxStat API for dataset `FIQ02` (Exchequer Account Historical Series)
2. Returns data (from 1975 to 1997) in JSON-stat 2.0 format 
3. Saves the response to `cso.json` with indented formatting for readability

### How to run
```bash
jupyter nbconvert --to notebook --execute assignment03-cso.ipynb
```
Or open in VS Code and click **Run All**.

### Output
A file called `cso.json` created in the same directory.

## Assignment 4 - GitHub API
**Files:** `assignment04-github.py`, `getrepoinformation.py`

### Description
Two programs that interact with the GitHub RESTful API:

1. **`assignment04-github.py`** — Reads a file from a public GitHub repository, replaces all instances of "Andrew" with "Orla", and pushes the updated file back to Orla's (seasid3) private repository using a personal access token.

2. **`getrepoinformation.py`** — retrieves the contents of Andrew's course material repository. When run, it generates a file called `wsaa-code.json` in the same directory.

### How it works - assignment04-github.py
1. Reads `bookapidao.py` from [andrewbeattycourseware/wsaa-courseware](https://github.com/andrewbeattycourseware/wsaa-courseware) (public repo, no token needed)
2. Replaces all instances of "Andrew" (capital "A") with "Orla". As this is case sensitive, lowercase instances - such as "andrewbeatty1" in the URL are not replaced
3. Checks if the file already exists in the destination repo and retrieves its SHA if it does
4. Pushes the updated file to the destination repo using the GitHub API

### How it works - getrepoinformation.py
1. Calls the GitHub API to get the contents of Andrew's course material repo
2. When run, generates `wsaa-code.json` in the same directory

### Security
This program uses a `config.py` file to store sensitive credentials. This file is listed in `.gitignore` and is never pushed to GitHub.

`config.py` contains:
```python
GitHub_token      = "your_token_here"
Destination_owner = "your_github_username"
Destination_repo  = "your_repo_name"
Destination_file  = "path/to/output/file.py"
```

### Files in this folder
| File | Description |
|------|-------------|
| `assignment04-github.py` | Main program - reads `bookapidao.py`, replaces "Andrew" with "Orla" and pushes to repo |
| `getrepoinformation.py` | Retrieves repo contents - generates `wsaa-code.json` when run |
| `config.py` | Stores credentials - never pushed to GitHub |
| `__pycache__/` | Auto-generated by Python - can be ignored |

### How to run
```bash
python assignment04-github.py
python getrepoinformation.py
```

### Expected output - assignment04-github.py

    File read successfully.
    Name replaced successfully.
    File successfully pushed to my repo.

### Expected output - getrepoinformation.py

    200

A file called `wsaa-code.json` will be created in the same directory.
